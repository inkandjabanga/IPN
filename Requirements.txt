Instant Payments Namibia (IPN) Developer Integration Challenge Senior Specialist – Developer
1. Overview
As part of the screening stage of the recruitment process, candidates are required to complete a short development challenge designed to demonstrate practical software development and system integration capabilities.
The objective of this challenge is to build a simple application that captures and submits a Person-to-Person (P2P) payment request, using the Mock API Specification provided.
2. Application Requirements
Create a simple application that allows a user to initiate a Person-to-Person (P2P) payment.
The application should:
• Provide a user interface screen that allows a user to capture the required payment details.
• Construct and submit the payment request according to the Mock API Specification.
• Perform basic validation of user input before submitting the request and prevent submission if required fields are invalid.
• Handle the API response and display the transaction outcome to the user.
Once a transaction is processed, the application should display:
• Transaction status (success or failure)
• Transaction ID (if applicable)
• Client transaction reference
• Response message returned by the API
Candidates may simulate or locally implement the API endpoint described in the Mock API Specification if needed.
Candidates may use any technology stack, framework, or development tools, including AI-assisted tools, but submissions should reflect your own understanding and approach to the problem.
Clear structure, error handling, and thoughtful design will be viewed favourably.
3. Submission Requirements
Please submit the following:
• Source code for the solution
• Screenshots of the application interface, including:
o the form used to capture payment details
o the displayed transaction result
• A short README describing:
o Setup instructions
o Any assumptions made
o How the integration was implemented
4. Submission
Please submit your solution by email no later than Monday at 10h00.
Your email should include:
• A link to your source code repository
• The README document
• Screenshots of the application interface
Please ensure that the repository is accessible to the review team, either by making it public or by granting appropriate access. Candidates may also attach a compressed (ZIP) copy of the source code as a backup.
Mock API Specification
P2P Payment Integration
1. Purpose
This document provides a simplified API specification used for the Developer Integration Challenge.
It is intended solely for recruitment screening purposes and does not represent the actual IPN production interface.
2. API Endpoint
Method: POST
Endpoint: /api/p2p-payment
Candidates may host or simulate this endpoint locally.
